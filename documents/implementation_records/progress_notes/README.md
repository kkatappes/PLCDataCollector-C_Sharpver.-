# 進捗メモ・課題点 (Progress Notes)

このフォルダーには、実装進捗の記録、課題・問題点の整理、今後の計画・アイデアなどを保存します。

## 記録の目的

- **進捗の可視化**: 実装の進み具合と完了予定の管理
- **課題の追跡**: 発生した問題と解決状況の継続管理
- **アイデアの蓄積**: 実装中に思いついた改善案や新機能アイデア
- **振り返り**: 定期的な進捗レビューと改善点の抽出
- **計画調整**: スケジュールや優先度の見直し記録

## ファイル命名規則とカテゴリ

### 📁 カテゴリ別フォルダー構成
```
progress_notes/
├── daily_progress/      # 日次進捗記録
├── weekly_reviews/      # 週次振り返り
├── issues_tracking/     # 課題・問題追跡
├── ideas_backlog/       # アイデア・改善案
├── milestone_reports/   # マイルストーン報告
└── retrospectives/      # 振り返り・改善記録
```

### 📄 ファイル命名規則

#### 日次進捗記録
```
{YYYY-MM-DD}_日次進捗.md
例: 2024-10-28_日次進捗.md
```

#### 週次振り返り
```
{YYYY-WW}_週次振り返り.md
例: 2024-W43_週次振り返り.md
```

#### 課題追跡
```
{YYYY-MM-DD}_課題_{課題ID}_{課題概要}.md
例: 2024-10-28_課題_001_TCP接続安定性.md
```

#### アイデアメモ
```
{YYYY-MM-DD}_アイデア_{カテゴリ}_{概要}.md
例: 2024-10-28_アイデア_パフォーマンス_並列処理.md
```

## 🛠️ 管理・活用方法

### 📅 定期的な記録習慣
- **毎日**: 日次進捗記録 (作業終了時)
- **毎週**: 週次振り返り (金曜日午後)
- **随時**: 課題・アイデア記録 (発生時)
- **月次**: マイルストーン振り返り
- **🚨 学習資料作成時**: 実装チェックリスト同時更新

### 🔍 分析・改善活動
- **トレンド分析**: 進捗パターンの把握
- **課題分析**: よくある問題の特定
- **効率化**: 生産性向上の施策検討
- **目標調整**: 現実的なペース設定

### 📊 可視化・共有
- **進捗グラフ**: 視覚的な進捗把握
- **課題ダッシュボード**: 課題の優先度管理
- **成果報告**: 定期的な成果共有

## 🎯 効果的な活用のコツ

### 継続するために
- **簡潔に**: 完璧を求めすぎず、要点を記録
- **習慣化**: 決まった時間に記録
- **テンプレート活用**: 毎回同じ形式で記録
- **振り返り**: 記録を見返して改善

### 質を高めるために
- **具体的に**: 抽象的でなく具体的な記録
- **数値化**: 感覚的でなく測定可能な指標
- **客観視**: 感情的でなく事実ベースの記録
- **アクション指向**: 記録だけでなく改善行動も
