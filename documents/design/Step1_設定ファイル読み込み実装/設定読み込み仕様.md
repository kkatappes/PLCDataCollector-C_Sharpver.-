# 設定ファイル読み込み仕様
一つのファイルとして実行ファイルパッケージ化したアプリケーションと同じフォルダ内にあるエクセルファイル(.xslx)をすべて読み込んで内容を抽出する

# 設定ファイル仕様
設定ファイルはexcelファイルから読み込み実施
excel内の各設定情報記載は下記の通り

## 設定情報
### タブ名
・settings

### 設定項目-セル紐づけ
・PLCのIPアドレス：B8
・PLCのポート：B9
・データ取得周期：B11
・デバイス名：B12
・データ保存先パス：B13

## データ収集デバイス設定
### タブ名
・データ収集デバイス

### データ収集デバイス項目-セル紐づけ
各項目は1行目ラベルのため、2行目から順に下に進んで空白行まで読み込み
・項目名：A列
・デバイスコード：B列
・デバイス番号：C列
・桁数：D列
・単位：E列

# Step7で出力するファイルのjson構造
設定項目,データ収集デバイスの情報と最終出力ファイルのキーを紐づける

{
  "source": {
    "plcModel": ,
    "ipAddress": ,
    "port": 
  },
  "timestamp": {
    "local": 
  },
  "items": [
    {
      "name": ,
      "device": {
        "code": ,
        "number": 
      },
      "digits": ,
      "unit": ,
      "value": 
    },
    {
      "name": ,
      "device": {
        "code": ,
        "number": 
      },
      "digits": ,
      "unit": ,
      "value": 
    }
  ]
}

・source.plcModel: string（必須）
・source.ipAddress: string（必須、IPv4/IPv6書式）
・source.port: integer（必須、1–65535）
・timestamp.local: string（必須、ISO 8601ローカル時刻、タイムゾーンオフセット付き）
・items: array（必須、要素はオブジェクト）
  - items[].name: string（必須）
  - items[].device.code: string（必須）
  - items[].device.number: string（必須）
  - items[].digits: integer（必須、1以上）
  - items[].unit: string（必須、"bit", "word", "dword"）
  - items[].value: string または number
    - unit が "bit" のとき: 0/1（単一値）
    - unit が "word"/"dword" のとき: string, int16/int32, uint16/uint32, float16/float32（単一値）

## 設定項目,データ収集デバイスとキーの紐付き
・PLCのIPアドレス：ipAddress
・PLCのポート：port
・デバイス名：plcModel
・項目名：name
・デバイスコード：code
・デバイス番号：number
・桁数：digits
・単位：unit
